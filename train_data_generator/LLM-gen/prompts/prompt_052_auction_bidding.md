# Процесс проведения электронного аукциона

Организатор публикует лот с описанием и стартовой ценой, участники регистрируются и вносят обеспечительный платеж. В назначенное время начинается торговая сессия, участники делают ставки с автоматическим продлением при ставке в последние минуты. Система проверяет валидность ставок и блокирует средства. После завершения победитель получает уведомление и заключает договор, остальным возвращается обеспечение. При отказе победителя лот передается следующему участнику. Используй 3 пула (Участники, Система аукциона, Организатор), Event-Based Gateway для обработки ставок, подпроцесс с циклом для торговой сессии, Timer Event для автопродления и завершения, Compensation Event для возврата обеспечения.
