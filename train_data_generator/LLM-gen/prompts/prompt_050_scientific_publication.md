# Процесс публикации научной статьи в журнале

Автор отправляет статью в редакцию, редактор проверяет соответствие тематике и отправляет на рецензирование двум независимым экспертам. Рецензенты параллельно оценивают работу и дают заключение (принять, доработать, отклонить). При необходимости доработки автор вносит правки и отправляет повторно. После принятия статья проходит корректуру, верстку и публикуется онлайн с присвоением DOI. Используй 4 пула (Автор, Редакция, Рецензент 1, Рецензент 2), Parallel Gateway для параллельного рецензирования, Complex Gateway для объединения решений рецензентов, подпроцесс с циклом для итераций доработки, Timer Boundary Event для контроля сроков рецензирования.
