# Процесс обработки транзакции в блокчейне

Пользователь инициирует транзакцию с указанием получателя и суммы, кошелек создает и подписывает транзакцию приватным ключом. Транзакция отправляется в mempool, где майнеры выбирают её для включения в блок на основе комиссии. Параллельно несколько майнеров соревнуются в решении криптографической задачи. Победивший майнер публикует блок, другие ноды валидируют блок и добавляют в цепь. После N подтверждений транзакция считается финализированной. Используй 4 пула (Отправитель, Сеть, Майнеры, Валидаторы), Event-Based Gateway для выбора транзакции майнером, Parallel Gateway для конкурентного майнинга, подпроцесс для консенсуса, Multiple Instance для параллельной валидации нодами.
