# Процесс выполнения квантовых вычислений

Пользователь отправляет квантовую схему (circuit) через облачный API, система проводит валидацию синтаксиса и оптимизацию схемы под топологию конкретного квантового процессора. Параллельно проверяется доступность квантового компьютера и резервируется время выполнения в очереди задач. Перед запуском система калибрует кубиты, измеряет уровень шума и корректирует параметры гейтов. Квантовая схема транспилируется в низкоуровневые инструкции с учетом связности кубитов и применением error mitigation техник. Задача выполняется множество раз (shots) для получения статистически значимых результатов, параллельно мониторится декогеренция и ошибки. Результаты измерений постобрабатываются с применением quantum error correction, агрегируются и возвращаются пользователю вместе с метриками качества (fidelity, gate errors). При критических ошибках задача автоматически перезапускается на резервном процессоре. Используй 6 лейнов (API Gateway, Оптимизатор, Калибровка, Квантовый процессор, Error Correction, Постобработка), подпроцесс с циклом для множественных shots, Parallel Gateway для параллельных измерений, XOR Gateway для проверки качества, Error Boundary Event для сбоев оборудования, Compensation Event для перезапуска.
