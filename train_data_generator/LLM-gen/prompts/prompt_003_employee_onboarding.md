# Процесс адаптации нового сотрудника

Создай BPMN диаграмму процесса адаптации (onboarding) нового сотрудника в компании.

## Основные участники:
- HR отдел
- IT отдел
- Руководитель подразделения
- Новый сотрудник

## Описание процесса:
1. HR получает информацию о выходе нового сотрудника на работу
2. HR создает личное дело и заводит сотрудника в систему
3. Параллельно запускаются три процесса:
   - IT отдел создает учетные записи и настраивает рабочее место
   - HR готовит welcome-пакет и документы для подписания
   - Руководитель назначает наставника и планирует первую неделю
4. IT отдел:
   - Создает корпоративную почту
   - Настраивает доступы к системам
   - Подготавливает компьютер и оборудование
5. HR отдел:
   - Готовит трудовой договор
   - Оформляет пропуск
   - Подготавливает welcome-пакет с брендированными материалами
6. Руководитель:
   - Назначает наставника
   - Составляет план адаптации на 3 месяца
   - Планирует встречи с командой
7. В первый рабочий день:
   - Сотрудник приходит в офис
   - HR проводит вводный инструктаж
   - Сотрудник подписывает документы
   - IT выдает оборудование и учетные данные
   - Руководитель знакомит с командой и наставником
8. В течение первой недели наставник проводит обучение
9. Через месяц - промежуточная встреча с руководителем для обратной связи
10. Через 3 месяца - финальная оценка прохождения испытательного срока
11. Если оценка положительная - сотрудник успешно завершает адаптацию
12. Если оценка отрицательная - принимается решение о продлении испытательного срока или увольнении

## Требования:
- Используй 1 пул с 4 лейнами (HR, IT, Руководитель, Сотрудник)
- Используй Parallel Gateway для параллельной подготовки разными отделами
- Добавь подпроцесс для процесса обучения в первую неделю
- Используй Timer Event для промежуточных встреч (через месяц и через 3 месяца)
- Используй XOR Gateway для принятия решения по итогам испытательного срока
- Добавь Message Event для уведомлений между участниками
