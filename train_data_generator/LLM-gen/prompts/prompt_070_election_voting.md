# Процесс электронного голосования на выборах

Избиратель проходит аутентификацию через госуслуги с многофакторной проверкой (биометрия, SMS, токен), система проверяет право голоса и отсутствие повторного голосования. После успешной аутентификации генерируется уникальный анонимный токен для обеспечения тайны голосования. Избиратель выбирает кандидата, система шифрует бюллетень и отправляет в распределенный блокчейн для защиты от фальсификаций. Параллельно несколько независимых серверов валидируют транзакцию и добавляют в цепь с цифровыми подписями. Избиратель получает криптографическое подтверждение с возможностью проверки учета голоса без раскрытия выбора. После закрытия участков запускается параллельный подсчет голосов на множестве нод с взаимной верификацией результатов. Наблюдатели от партий получают доступ к аудит-логам для проверки честности. Используй 6 пулов (Избиратель, Система аутентификации, Блокчейн, Валидаторы, Подсчет голосов, Наблюдатели), Parallel Gateway для параллельной валидации, подпроцесс для шифрования, XOR Gateway для проверки права голоса, Multiple Instance для распределенного подсчета, Signal Event для синхронизации закрытия участков.
