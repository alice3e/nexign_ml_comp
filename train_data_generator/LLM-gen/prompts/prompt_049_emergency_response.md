# Процесс реагирования на чрезвычайную ситуацию

Диспетчер получает вызов о ЧС, определяет тип и масштаб происшествия, параллельно отправляет бригады (пожарные, скорая, полиция) в зависимости от ситуации. На месте координатор оценивает обстановку, при необходимости запрашивает дополнительные силы или эвакуацию. Все действия документируются в реальном времени с отправкой отчетов в штаб. Используй 5 лейнов (Диспетчер, Пожарные, Скорая помощь, Полиция, Координатор), Event-Based Gateway для определения типа ЧС, Parallel Gateway для одновременной отправки служб, подпроцесс Ad-Hoc для действий на месте, Escalation Event для запроса подкрепления, Timer Event для контроля времени реагирования.
