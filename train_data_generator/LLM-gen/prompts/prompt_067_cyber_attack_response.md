# Процесс реагирования на кибератаку

SIEM система детектирует аномальную активность (множественные неудачные попытки входа, подозрительный трафик, изменение критических файлов), автоматически генерирует инцидент и оповещает SOC аналитика. Аналитик проводит первичную классификацию угрозы, при подтверждении атаки эскалирует до уровня Incident Response Team. Параллельно запускаются автоматические меры защиты: блокировка подозрительных IP, изоляция скомпрометированных хостов, сброс сессий. Команда реагирования проводит форензик-анализ, определяет вектор атаки и масштаб компрометации, собирает артефакты для расследования. При обнаружении ransomware немедленно изолируются все системы и запускается восстановление из backup. Параллельно юристы готовят уведомления регуляторам, PR-служба готовит коммуникацию. После устранения угрозы проводится post-mortem анализ и обновление правил защиты. Используй 7 лейнов (SIEM, SOC, IR Team, Forensics, IT Operations, Legal, Management), Event-Based Gateway для классификации угрозы, Parallel Gateway для параллельных защитных мер, подпроцесс для форензик-анализа, Escalation Event для критических инцидентов, Compensation Event для восстановления систем.
