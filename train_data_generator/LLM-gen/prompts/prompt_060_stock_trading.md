# Процесс исполнения биржевой заявки

Трейдер размещает заявку на покупку/продажу акций через торговый терминал, брокер проверяет наличие средств или ценных бумаг на счете. Заявка отправляется на биржу и попадает в стакан заявок, торговая система сопоставляет встречные заявки по цене и времени. При совпадении условий происходит сделка, параллельно обновляются позиции обеих сторон и рассчитываются комиссии. Клиринговая палата проводит расчеты и переводит средства между счетами с T+2 settlement. Депозитарий регистрирует переход прав собственности на акции. При частичном исполнении заявка остается в стакане до полного исполнения или отмены. Используй 5 пулов (Трейдер, Брокер, Биржа, Клиринг, Депозитарий), Event-Based Gateway для сопоставления заявок, Parallel Gateway для параллельных расчетов, подпроцесс с циклом для частичного исполнения, Timer Event для settlement period, Compensation Event для отмены сделки при ошибке.
