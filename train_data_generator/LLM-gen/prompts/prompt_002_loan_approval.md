# Процесс одобрения кредита в банке

Создай BPMN диаграмму процесса рассмотрения и одобрения кредитной заявки в банке.

## Основные участники:
- Клиент
- Менеджер по кредитам
- Система скоринга
- Кредитный комитет

## Описание процесса:
1. Клиент подает заявку на кредит с указанием суммы и срока
2. Менеджер проверяет полноту документов
3. Если документы неполные - запрашивает дополнительные документы у клиента
4. Если документы полные - передает заявку в систему скоринга
5. Система автоматически проверяет кредитную историю и рассчитывает скоринговый балл
6. Параллельно система проверяет:
   - Наличие задолженностей
   - Уровень дохода
   - Стабильность занятости
7. На основе скорингового балла принимается решение:
   - Если балл высокий (>700) - автоматическое одобрение
   - Если балл средний (500-700) - передача на рассмотрение кредитному комитету
   - Если балл низкий (<500) - автоматический отказ
8. Кредитный комитет рассматривает заявку и принимает решение (одобрить/отклонить)
9. При одобрении:
   - Менеджер готовит кредитный договор
   - Клиент подписывает договор
   - Система выдает кредит
10. При отказе - клиент получает уведомление с причинами отказа
11. Процесс завершается

## Требования:
- Используй 2-3 пула (Клиент, Банк с лейнами для менеджера и комитета, Система)
- Добавь подпроцесс для проверки документов
- Используй XOR Gateway для принятия решений на основе скорингового балла
- Используй Parallel Gateway для параллельных проверок
- Добавь Timer Event для ограничения времени рассмотрения заявки комитетом (3 дня)
- Используй Boundary Event для обработки таймаута
