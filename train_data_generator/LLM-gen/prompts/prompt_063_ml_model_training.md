# Процесс обучения ML модели в production

Data Engineer загружает данные из источников, проводит валидацию качества и предобработку (очистка, нормализация, feature engineering). Параллельно Data Scientist выбирает архитектуру модели и гиперпараметры для экспериментов. Запускается распределенное обучение на GPU кластере с автоматическим логированием метрик в MLflow. По завершении каждой эпохи проверяется early stopping условие, лучшая модель сохраняется. После обучения проводится валидация на тестовой выборке, A/B тестирование на production трафике и мониторинг drift метрик. При деградации качества автоматически откатывается на предыдущую версию. Используй 6 лейнов (Data Pipeline, Training, Validation, MLOps, Monitoring, Production), подпроцесс с циклом для эпох обучения, Parallel Gateway для распределенных вычислений, XOR Gateway для early stopping, Error Event для сбоев обучения, Compensation Event для rollback модели.
