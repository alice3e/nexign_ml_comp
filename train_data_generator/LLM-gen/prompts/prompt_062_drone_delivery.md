# Процесс доставки посылки дроном

Система получает заказ и рассчитывает оптимальный маршрут с учетом погоды, зон ограничения полетов и препятствий. Дрон автоматически загружается посылкой, проходит предполетную диагностику систем и взлетает. В полете постоянно корректируется траектория на основе данных GPS, камер и датчиков препятствий, параллельно мониторится уровень заряда батареи. При критическом разряде или неблагоприятной погоде дрон экстренно возвращается на базу. По прибытии дрон сканирует зону посадки, опускается и отправляет уведомление получателю с кодом разблокировки контейнера. Используй 5 лейнов (Диспетчерская, Дрон, Навигация, Мониторинг, Клиент), подпроцесс для планирования маршрута, Parallel Gateway для параллельного мониторинга систем, Error Boundary Event для аварийных ситуаций, XOR Gateway для принятия решений в полете, Signal Event для коммуникации с базой.
