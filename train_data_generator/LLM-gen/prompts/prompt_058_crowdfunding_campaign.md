# Процесс краудфандинговой кампании

Автор создает проект с описанием, целевой суммой и вознаграждениями для спонсоров, платформа проводит модерацию на соответствие правилам. После одобрения запускается кампания на фиксированный срок, спонсоры делают взносы выбирая уровень вознаграждения. Система отслеживает прогресс сбора средств и отправляет обновления подписчикам. При достижении цели до дедлайна средства списываются со счетов спонсоров и передаются автору за вычетом комиссии, начинается выполнение проекта. Если цель не достигнута - все взносы возвращаются спонсорам. Автор регулярно публикует отчеты о прогрессе и отправляет вознаграждения. Используй 3 пула (Автор, Платформа, Спонсоры), Timer Event для дедлайна кампании, XOR Gateway для проверки достижения цели, подпроцесс с циклом для публикации обновлений, Compensation Event для возврата средств, Multiple Instance для обработки множественных спонсоров.
