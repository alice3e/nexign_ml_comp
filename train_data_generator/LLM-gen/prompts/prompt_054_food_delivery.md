# Процесс доставки еды через агрегатор

Клиент выбирает ресторан и блюда, оформляет заказ с указанием адреса и времени доставки. Система проверяет доступность ресторана и передает заказ, ресторан подтверждает и начинает готовить. Параллельно система ищет свободного курьера поблизости, курьер принимает заказ и едет в ресторан. После получения заказа курьер доставляет клиенту, клиент оплачивает и оценивает заказ. При отмене заказа клиентом или рестораном запускается процесс возврата средств. Используй 4 пула (Клиент, Платформа, Ресторан, Курьер), Parallel Gateway для параллельного поиска курьера и приготовления, XOR Gateway для обработки отмены, Signal Event для уведомления курьера о готовности заказа, Compensation Event для возврата оплаты.
